Features

Everything you need.
Nothing you don't.

Oja packs routing, reactive state, and a template system into a single small file. No config, no CLI, no toolchain — just drop it in and go.

Zero Build Setup

Drop a single script tag. No npm install, no webpack config, no Babel. Works in any browser instantly.

Reactive State

Proxy-based reactivity keeps your UI in sync with state. No boilerplate signals, just plain JavaScript objects.

Lightweight Router

Hash or history API routing with nested routes, params, guards, and lazy loading — in under 1KB.

Template System

HTML-first templates with expressions, loops, conditionals, and slots. Familiar syntax, zero learning curve.

Multi-App Support

Run multiple independent SPA islands on the same page. Perfect for micro-frontends and portals.

Component Model

Define reusable components with scoped styles, local state, and lifecycle hooks — all in pure JS.

Interactive Demo

See Oja in action.

Real code. Real patterns. Toggle between examples and watch the fake browser react. 

// Define your routes
import { createRouter } from './oja.js'

const router = createRouter({
  routes: [
    {
      path: '/',
      component: HomeView,
      title: 'Home'
    },
    {
      path: '/about',
      component: AboutView
    },
    {
      path: '/counter',
      component: CounterView
    }
  ]
})

// Mount the app
router.mount('#app')

// Navigate programmatically
router.push('/about')

// Reactive state with Proxy
import { reactive, watch } from './oja.js'

const state = reactive({
  count: 0,
  user: { name: 'Ada' }
})

// Auto-updates DOM on change
watch(state, (key, val) => {
  console.log(`${key} changed to`, val)
})

// Simple mutation triggers UI update
state.count++
state.user.name = 'Kofi'

// Computed values
const doubled = () => state.count * 2

<!-- Oja template syntax -->
<template id="home-view">

  <h1>Hello, {{ user.name }}</h1>

  <!-- Conditional rendering -->
  <p oja-if="count > 5">
    Count is getting high!
  </p>

  <!-- Loop rendering -->
  <ul>
    <li oja-for="item in items">
      {{ item.label }}
    </li>
  </ul>

  <!-- Event binding -->
  <button oja-on:click="increment">
    Count: {{ count }}
  </button>

</template>

How It Works

From idea to SPA in 3 steps.

No scaffolding. No ceremony. Oja gets out of your way so you can ship.

01

Define Routes

Map URL paths to view components. Oja handles history, hash routing, and nested layouts.

createRouter({ routes: [...] })
02

Attach Templates

Write standard HTML templates with Oja directives. No JSX, no virtual DOM, just markup.

<div oja-for="item in list">
03

Add Reactivity

Wrap your data with reactive(). Any mutation auto-updates the DOM. That's it.

const state = reactive({ count: 0 })

How Oja stacks up.

We respect React and Vue — they're great. But not every project needs a build pipeline.

No build step ✓ Yes ✗ No ~ CDN only
Bundle size ~3KB ~45KB ~33KB
Routing built-in ✓ Yes ✗ External ~ External
Reactive state ✓ Yes ✓ Yes ✓ Yes
Template syntax HTML-first JSX HTML-first
Multi-app SPA ✓ Native ~ Manual ~ Manual
Zero dependencies ✓ Yes ✗ No ✗ No

Why choose Oja?

Open index.html. Add one script tag. You're done — no npm, no config.
3KB vs 45KB means faster loads, better Lighthouse scores, and simpler deploys.
Multi-app architecture lets you scope SPAs to page sections — ideal for portals and dashboards.
HTML-first templates means designers and devs can collaborate without learning JSX.
Perfect for prototypes, dev tools, internal dashboards — anything where a full framework is overkill.
